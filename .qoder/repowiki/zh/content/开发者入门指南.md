## 简介
本指南旨在帮助新手开发者快速搭建并运行本项目。项目包含基于Spring Boot的后端服务和基于React的前端应用，通过本指南可完成从零开始的完整部署流程。

## 项目结构
项目采用前后端分离架构，主要目录结构如下：
- `frontend/`：前端React应用
- `src/`：后端Spring Boot源码
- `pom.xml`：Maven构建配置文件
- `README.md`：项目说明文件

## 环境准备
在开始之前，请确保已安装以下基础环境：
- **Node.js**（建议v16以上版本）：用于运行前端应用
- **npm**：Node.js包管理工具
- **Java 8或以上版本**：后端服务运行环境
- **Maven**：Java项目构建工具

可通过以下命令验证环境是否正确安装：
```bash
node --version
npm --version
java -version
mvn --version
```

## 依赖安装
### 前端依赖安装

## 服务启动
### 启动后端服务
在项目根目录执行以下命令启动Spring Boot应用：
```bash
mvn spring-boot:run
```
服务将启动在端口9090（由`application.properties`配置），提供REST API接口。

### 启动前端服务
打开新终端窗口，进入frontend目录并启动前端：
```bash
cd frontend
npm start
```
前端开发服务器将启动在`http://localhost:3000`。

### 注意事项
- **必须同时运行前后端服务**
- 确保端口`9090`（后端）和`3000`（前端）未被其他程序占用
- 若端口被占用，可修改`application.properties`中的`server.port`值

## 快速测试流程
完成服务启动后，可按照以下步骤测试系统功能：

### 1. 创建订单
- 在前端界面填写订单信息（如订单号、商品名称、数量等）
- 点击"创建订单"按钮
- 系统发送POST请求至`/api/orders`接口

### 2. 查询订单
- 在查询输入框中输入订单号
- 点击"查询"按钮
- 系统发送GET请求至`/api/orders/{id}`接口
- 查看返回的订单详情

### 3. 更新订单状态
- 查询到订单后，点击"更新状态"按钮
- 选择新的状态（如"已发货"、"已完成"等）
- 系统发送PUT请求至`/api/orders/{id}`接口

### 4. 删除订单
- 选择需要删除的订单
- 点击"删除"按钮
- 系统发送DELETE请求至`/api/orders/{id}`接口
- 确认订单已从列表中移除

## 常见问题排查
### 问题1：前端页面无法加载
**可能原因**：后端服务未启动  
**解决方案**：
1. 检查后端是否已执行`mvn spring-boot:run`
2. 查看终端是否有启动成功日志
3. 确认端口9090未被占用：`lsof -i :9090`

### 问题2：API请求失败（404或连接拒绝）
**可能原因**：前后端端口配置不匹配  
**解决方案**：
1. 确认后端`application.properties`中`server.port=9090`
2. 检查前端`orderService.js`中API基础URL是否为`http://localhost:9090`

### 问题3：npm install失败
**可能原因**：网络问题导致包下载失败  
**解决方案**：
1. 使用国内镜像源：`npm config set registry https://registry.npmmirror.com`
2. 清除缓存后重试：`npm cache clean --force`

### 问题4：Maven编译失败
**可能原因**：依赖下载失败  
**解决方案**：
1. 检查网络连接
2. 使用Maven镜像源（在`settings.xml`中配置阿里云镜像）
